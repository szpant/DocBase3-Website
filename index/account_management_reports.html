Exception Report of Payment Requests Acquitted by A/C (P1AA9700)
Exception Report of Payment Requests Acquitted by A/C (P1AA9700)
    What it isThe report lists all payment requests where the customer account from which the payment was made is not the same product owner.
    Frequency
    • Daily for the previous days transactions. A nil report will be produced if no reportable transactions occurred the previous day.
    • On demand
    Report Details
    Information displayed on the report includes
    Information displayed on the report includes
      • Date of payment
      • User ID
      • Payment request type and reference number
      • Payment request amount
      • Product type
      • Amount of the transaction
      • Name of the customer (individual or organisation)
      • Customer account number
    Checking GuidelinesWhen actioning this report the investigating officer is to confirm that the funds were used for the intended purpose. For example: If a customer has monies in their account for a practical driving test, these funds should not be used to pay for a registration renewal.

      Note:When completing this report, ensure the purpose of each action appearing on the report is noted on the Action line. For example; renew rego from PPS (planned payment scheme), dealer/agency processing

    RulesThe report must be printed and actioned daily unless it is a Nil Report. Nil Reports are not printed, they are kept in the Customer Service Centre/Issuing Agency TRAILS reports mailbox.

    Report items should only be added to the follow up register when corrective action needs to be taken. That is the action taken to generate the report item was incorrect.

    Standard business processes such as processing agency work by invoicing the money to the customer and then paying from the customers account are not incorrect and do not need to be recorded in the follow up register.

    Corrective action should be completed within 14 days.
Daily Customer Account Report (DR/CR Report) (P1AA6200)
Daily Customer Account Report (DR/CR Report) (P1AA6200)
    What it isThis report lists accounts that have a debit or credit balance. It is produced by location and the DR/CR balance appears on the report of the CSC that took the most recent action.
    Frequency Daily for the previous days transactions.
    Report Details
    Information displayed on the report includes
    Information displayed on the report includes
      • CSC Office
      • Date and time
      • Customer Reference Number and Name
      • User ID of the person who made the most recent transaction
      • Current balance
      • Remarks and Date field to note action taken
    Checking GuidelinesWhen actioning this report the Checking Officer is to confirm:
    • For a credit: ensure the purpose of each credit appearing on the report is noted on the Action line. For example; payment for PDT, Planned payment scheme, cancellation of rego.
    • For a debit: that the transaction that raised the debt is correct and ensure the purpose of each debit appearing on the report is noted on the Action line. For example; agency processing, transfer payment not paid.

    The Checking Officer is only required to check the portion of any DR/CR that was created at their location.
    RulesThe report must be printed and actioned daily unless it is a Nil Report. Nil Reports are not printed, they are kept in the Customer Service Centre/Issuing Agency TRAILS reports mailbox.

    Report items should only be added to the follow up register when corrective action needs to be taken. That is the action taken to generate the report item was incorrect.

    Standard business processes such as suspending early renewal payment are not incorrect and do not need to be recorded in the follow up register.

    Corrective action should be completed within 14 days.

List of Operational Account Balances (AAR5BE)
List of Operational Account Balances (AAR5BE)
    What it isThe report is produced to identify daily balances in an operators operational accounts (OPCASHBK, OPFLOAT, OPOVERS and OPUNDERS).
    FrequencyDaily for the previous days balances
    Report Details
    Information displayed on the report includes
    Information displayed on the report includes
      • Cumulative totals of the locations
      • Balances of employee operational accounts
      • Operators:
        • Account Name
        • Account Number
        • Account Type
        • Current Balance
    RulesThe report must be printed daily and actioned by:
    • Any operators appearing on the report who no longer operate at that location must have their user profile updated to remove them from the business unit.
    • If no corrective action required, no mark needs to be recorded on the report
    • If corrective action is required then the Follow Up Register number needs to be written on the first initial report showing the under/over >$5. The follow up number does not need to be written on any subsequent reports
    • For further action of unders/overs >$5 refer to the Adjustments Guide

    Standard business processes such as managing operator account when a TICA EFTPOS freeze has occurred are not incorrect if being correctly applied and therefore do not need to be recorded in the follow up register.

    Corrective action should be completed within 14 days.

List of Manual Adjustments (P1AA9300)
List of Manual Adjustments (P1AA9300)
    What it isThis report provides details of any manual financial adjustments.
    FrequencyDaily for the previous days adjustments. A nil report will be produced if no reportable transactions occurred the previous day.
    Report Details
    Information displayed on the report includes
    Information displayed on the report includes
      • Adjustment Date
      • User ID and name of the adjusting operator
      • Type of adjustment
      • Adjustment number
      • Adjustment amount
      • Text associated with the adjustment
    Checking GuidelinesThe following must be checked/considered when actioning the report where the answer is no correction action is required:
    • Was an adjustment the most appropriate way to manage the transaction?
    • If the adjustment was appropriate:
      • Were the correct adjustment type and accounts used?
      • Was appropriate text recorded in TICA?

      Note:When completing this report, ensure the purpose of each action appearing on the report is noted on the Action line. For example; operator under/over, agency processing.

    Action RequiredThe report must be printed and actioned daily unless it is a Nil Report. Nil Reports are not printed, they are kept in the Customer Service Centre/Issuing Agency TRAILS reports mailbox.

    Report items should only be added to the follow up register when corrective action needs to be taken. That is the action taken to generate the report item was incorrect and must be corrected.

    Where an adjustment relates to an automatic driver licence refund the Checking Officer must record the CRN next to the item on the report and note that it is an "Auto adjust for medical/deceased surrender"

    Standard business processes are not incorrect if being correctly applied and do not need to be recorded in the follow up register.

    Corrective action should be completed within 14 days.
