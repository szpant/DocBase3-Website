Administration Reports
Batch Address Validation
Batch Address Validation
    Description
    		This report identifies all addresses that were unable to be validated during the batch run.

    The purpose of this is to match the invalid addresses with valid address from the tables. When a match occurs the address will be marked as valid. If the address is unable to be matched, it will remain as invalid and be shown on the batch report.
    Details Produced
    • Sequence number
    • Address ID
    • Error
    • Address
    Frequency
    		This process should be run every three (3) months.

Miscellaneous Collections
Miscellaneous Collections
    Description
    		Identifies miscellaneous NON customer products that have either:
    • Variable fee products
    • Mandatory input of text (those products which automatically flow to the text screen (eg unregistered vehicle permits, buoy mooring fees)

    CSC Manager/Supervisor/Banking Officer is:
    • Ensure report is printed daily
    • Check that all funds have been applied to the correct accounts
    • Take corrective action if necessary (adjust payment to correct revenue account)
    • Sign, date and note adjustment number
    • File and retain reports
    Details Produced
    • Date and number of transaction
    • Amount
    • User ID and name
    • Explanatory text regarding collections
    Frequency
    		Daily
Practical Driving Test Booking Report
Practical Driving Test Booking Report
    Description
    		This report lists all practical driving test bookings that have a discrepancy.
    Details Produced
    • Date and time
    • Booking Location
    • Customer Reference, Product Type, Booking Date and Booking Time
    Checking Guidlines
    		CSC Manager/Supervisor/Driving Examiner is to:
    • Ensure report is printed
    • Each line item is to be investigated and actioned
    • Text is to be recorded next to each line item to confirm the action taken
      • Example: 12345678 PTC 01/07/2020 08:30 - no result entered. (DE Name) result entered. Action Finalised
    • Userid of actioning officer and date to be written on report
    • Report is to be filed as per current trails reports guidelines

    Note:If any line item cannot be actioned, or you are unsure of how to resolve the discrepancy, the report should be forwarded to your regional SBS Manager to review and action if required.
    Frequency
    		Report is generated when a discrepancy has occurred

Document Filing Summary Report
Document Filing Summary Report
    Description
    		The Document Filing Summary report lists all operators who have filing by product category.
    Details Produced
    • The report lists the UserIDs of operators who have filing for each product group.
    • The report is provided to assist CSCs in ensuring all operator filing is present when checking the daily filing and packaging filing to be sent to Iron Mountain
    • There is no requirement for CSCs to use this report.
    • The report does not have be printed.
    • The report does not have be retained.
    Frequency
    		Daily
Daily Medical Pre-requisite Verification Report
Daily Medical Pre-requisite Verification Report
    Description
    		This report is provided to assist checking officers determine if a medical certificate has been correctly recorded in TICA.

    It lists all instances where an MEDC Prerequisite has been passed in the Driver Licence Hub and displays details of the MCR Hub Add new report information recorded after the passing of the MEDC prerequisite and the current conditions on the driver licence.
    Details Produced
    • The report is provided to assist Checking Officers identify errors related to recording medical conditions in TICA
    • There is no requirement for CSCs to use this report.
    • The report does not have be printed.
    • The report does not have be retained.

    Guide to assist staff you use the Daily Medical Pre-requisite Verification Report:
    Guidelines - MPV
    Frequency
    		Daily if qualifying transactions processed.
Un-finalised Medical Condition Report
Un-finalised Medical Condition Report
    Description
    		This report shows medical condition reports that have been added to the MCR Hub and not finalised.

    As only medical condition reports related to current show cause actions should be un-finalised, the report enables Checking Officers to identify reports that have been incorrectly left un-finalised.
    Details Produced
    • Follow up Date
    • Customer Reference Number
    • Customer Details
    • Action Requiring Follow-up
    • Action Officer Details

    Show Cause actions initiated at the CSC will appear on the report until process is finalised. The report is to be Actioned on the day it is produced.
    Frequency
    		Daily if qualifying transactions processed.
