Driving Examinations
    Added a "Fail to Arrive" result on SBS in error and cannot change the result
    Added a "Fail to Arrive" result on SBS in error and cannot change the result
      How it occurred:
      An "Other" result in SBS creates an Auto forfeit of the booking in TRAILS and cannot be changed.

      To Confirm Problem:
      Check the TEST screen in TRAILS and ensure that the booking is auto forfeited.

      Solution
      (Refer procedure "Correct an Other result")

    Booking for MR class & arrived in LR vehicle. SBS won't allow change of class.
    Booking for MR class & arrived in LR vehicle. SBS won't allow change of class.
      How it occurred:
      A customer has been booked for an MR class test and has arrived in a LR class vehicle. SBS will not allow the class of test to be changed.

      SBS does not allow change of class between HR/MR to LR or visa versa.

      This is due to the fact that an LR class is a 1 slot test (45 min) and HR/MR tests are 2 slot tests (90 min).

      Solution
      (Refer procedure "Changing Classes from MR/HR to LR")

    An error occurs "No BOOKED Test Fee" when adding a result in SBS
    An error occurs "No BOOKED Test Fee" when adding a result in SBS
      This means there is no longer a booking in TRAILS.

      To Confirm Problem:
      Expand the TEST screen in TRAILS to establish what action has been taken to remove the booking in TRAILS.

      There are 3 possible causes:
      1. Status TERM - the credit action taken in TRAILS
      1. Status TERM - the credit action taken in TRAILS
        How it occurred:
        If the status of the booking is now TERM this means the credit action has been taken in TRAILS and payment has been credited to the customer account.

        Solution
        If the result being entered is "Fail to Arrive" and TRAILS shows that the credit action was taken two or more working day before the date of the booking, cancel the booking offline in SBS and no result needs to be added. The reason for this action is that it appears that the customer has contacted TMR to cancel the booking and request reimbursement of their test fee and the operator failed to cancel the booking in SBS.

        If the above is not the case then further investigation would be required to establish why this action was taken.

      2. Status FORF - auto forfeit has occurred
      2. Status FORF - auto forfeit has occurred
        How it occurred:
        If the booking has been "Auto forfeited" this may mean that a result has been entered on an old booking that should have been cancelled.

        Scenario:
        Scenario:
          Customer rings to reschedule their driving test booking. The SBS call centre currently have a procedure that if a booking cannot be rescheduled but the customer is entitled to reimbursement of their test fees the operator credits the booking on the TEST screen in TRAILS and then cancels the booking offline in SBS. They then make a new booking using the money that has been credited to the customers account.

          When a call centre operator credits the booking in TRAILS but fails to process the cancellation in SBS and then makes the new booking the old booking will still appear on the daily worksheet for that location. SBS allows the new booking as it only checks that a booking does not already exist for this class in TRAILS.

          Because it still appears on the daily worksheet for that location, when the customer does not arrive for the test (because they believe that the test has been cancelled) the driving examiner enters a "fail to arrive" result in SBS. SBS looks in TRAILS for a booking (SBS looks for a status of BOOK it does not look for date and time) this action auto forfeits the current booking.

        Solution
        The new booking will need to cancelled and rebooked before the new result can be entered. (Refer procedure "Correct an Other Result")

      3. Status PASS or FAIL - result processed on TRAILS not SBS
      3. Status PASS or FAIL - result processed on TRAILS not SBS
        If the result has already been recorded this may mean that the result was processed directly on TRAILS instead of correctly in SBS.

        To Confirm Problem:
        Check the PREQ screen in TRAILS and view the history. This should indicate the userid and location where the result was entered.

        Solution
        This result needs to be added offline in SBS and the operator advised of the correct procedure.

    Error occurs when printing Daily worksheet
    Error occurs when printing Daily worksheet
      The following errors occurred when trying to print Daily Worksheet:
      • "Unable to find result for returnCode =0 reasonCode =0 componentPrefix = because: null"
      How it occurred:
      This means that there is an error on one of the bookings at the location and date that the worksheet is being printed for.

      To Confirm Problem:
      Search on the weekly booking schedule search screen for this location and date. Select Slot Types Booked. (Refer Search of a booking procedure).

      The error "Unable to find result for returnCode =0 reasonCode =0 componentPrefix = because: null" is usually a booking that has been made for a customer with only one name in TRAILS.

      To Confirm Problem:
      Second level support may need to contact ISB with the information of the date, time and location of the slot where the error appears. ISB will let us know the CRN of the customer. If the error is not picked up until close to the time of the booking it may be easier to get the customers CRN when they come into the CSC centre for their test.

      Solution
      Once the CRN is known add something in the first name field in TRAILS for example, "=" symbol this allows the daily worksheet to be printed and a result to be entered in SBS. This can be removed once a result has been entered on SBS.

    Booking for vehicle inspection on daily worksheet but booking cancelled in TRAILS.
    Booking for vehicle inspection on daily worksheet but booking cancelled in TRAILS.
      How it occurred:
      This means that there has been a change on TRAILS to the registration that has caused the booking to be cancelled.

      To Confirm Problem:
      Check the registration history screen in TRAILS to establish what changes have been made since the inspection booking was made.
      There are 2 possible causes:
      1. Purpose of Use Changed
      1. Purpose of Use Changed
        How it occurred:
        The purpose of use should either be changed prior to the booking being made or after the inspection has been issued.

        Solution
        Cancel the booking offline in SBS and rebook exempt fee.

      2. There has been a transfer of registration.
      2. There has been a transfer of registration.
        How it occurred:
        Where there has been a transfer of registration the booking will be cancelled if the date the booking was processed is prior to the transfer date of the registration. This would indicate that the old owner made and paid for the booking. The SBS call centre gets a report on booking that are cancelled due to transfer of registration.

        If the transfer date is on or before the date the booking was processed the booking will transfer to the new owner as this indicates that the new owner made and paid for the booking.

        Solution
        The SBS call centre gets a report on booking that are cancelled due to transfer of registration. They normally contact the old owner to check if the booking is to be cancelled or transferred to the new owner. If the booking is to be transferred to be new owner, the booking should be cancelled in SBS offline and rebooked exempt fees.
