Override a Non-compliant Address (2996)

Applies to
Applies to
    The override function is used if an address is not found by TICA. This might arise for a number of reasons including (but not limited to):
    • The customer lives outside of an Australia Post delivery area and therefore their address details might not appear on the PAF.
    • The customer lives in a new street which has yet to appear on the current PAF.
    • An error exists in either the customer's address or in the PAF.

    In these circumstances, it will be necessary to enter a non-compliant address into TICA using the override function.
1
Ensure that non-compliant address details are valid
Ensure that non-compliant address details are valid
    A CSO should take steps to check an address provided by a customer to make it compliant. This should include:
    • Use of the drop-down menu in TICA
    • Request assistance from the customer to clarify their address details
    • Ask for any supporting documentation that the customer may have with them.

    Note:The address must contain a compliant suburb/ state/ postcode combination (at minimum).

    If the address still can't be ascertained after these steps, or the customer is disputing the address (that is allocated automatically by the system), a search of QLD Globe may be conducted to establish the correct address.

    The customer must produce supporting documents (eg. rates notice) if the non-compliant address can't be resolved by searching the Qld Globe website.

    Note:QLD Globe cannot be accessed through web browser 'Internet Explorer' therefore please use web browser 'Chrome' or 'Edge'.
2
Enter address on the system using the override function
Enter address on the system using the override function
    Enter address on TICA
    Enter address on TICA
      • Update customer address
      • When entering a non-compliant address on TICA it will raise a dialogue box, select the 'Override' push-button to override address.
