Non-Integrated EFTPOS Transactions (2507)

Non-Integrated EFTPOS refers to EFTPOS devices that require operator interaction and the device does not function through TICA for payments, refunds and cancellations.

A transaction is not to be finalised on an EFTPOS Terminal unless the relevant sections have been verified:
  • Purch/Refund
  • Approve/Decline
  • Amount and Signature (credit card)

For more information regarding acceptable bank card transactions Refer: Admin > 3. Money > Administration Money Guide > Payment Methods
Purchase Transaction
Purchase Transaction
    Terminal DisplayActions Required
    Prior to commencing EFTPOS transaction operator must determine if the customer is paying with a Visa or MasterCard branded card. If Yes, select the MCARD or VISA payment method on the TICA Make Payment Hub. This will generate the correct surcharge amount. This update amount must be confirmed with the customer before the EFTPOS transaction is to commence.
    Ready
    • Swipe the card through the card reader
    Select
    • Transaction Type
    • Use yellow keys to select transaction type from the three options displayed:
      • PUR (purchase)
      • CASH (cash out)*
      • REF (refund)
    * Note: TMR does not provide this service
    Enter
    • Amount
    • Enter $ amount into terminal (including the card surcharge if applicable.
    • Ask customer to confirm the amount
    • Press OK button
    Select
    • Account
    • Ask customer the required account type from the three options displayed
    • Use yellow keys to select the transaction type
      • CHQ (cheque account)
      • SAV (savings account)
      • CR (credit account)
    Enter
    • PIN
    • Customer verifies the amount (including the card surcharge if applicable)
    • Enter PIN
    • Press OK button
    Note:Bank cards do not always require a PIN to be entered.The operator should always be guided by the terminal of what authorisation is required.
    Processing
    • Terminal communicates with the CBA computer for authorisation
    ApprovedWhen terminal completes printing the receipt:
    • Check the details of the receipt
    • Press OK button
    • Circle PURCH or REFUND transaction type
    • Circle APPROVED of DECLINED message
    • Circle the Amount
    • Enter the details into TICA
    • Hand the duplicate receipt to the customer

    Note:If receipt does not print:
  • Refer to Printer Malfunction procedures
  • Do not re-enter the transactions

    If EFTPOS Lines are Down
    If EFTPOS Lines are Down
      If the EFTPOS lines are down the EFTPOS machine will store the transaction and forward it to the bank when the EFTPOS lines are operational.

      By re-entering the transaction / re-swiping the card through the EFTPOS machine, the customer will be charged multiple times.

Printer Malfunction
Printer Malfunction
    Check the printer is turned ON, confirmed by the green indication light.
    To turn ON press '1' on the printer touch pad.

    Printing Errors
    Printing Errors
      If a printing error occurs, there are two options:
      • Turn the printer ON
      • Replace paper
      • Reprint a duplicate receipt from the functions menu:
      • Press blue 'F' key on terminal keypad
      • Select '1. Paper/Statistics' option
      • Select '1 Duplicate Receipt' option
      • Switch the printer OFF
      • Press '0' on the touchpad
      • Use Card Transaction Record Vouchers until the printer is replaced

    Processing Transaction with printer OFF
    Processing Transaction with printer OFF
      On-Line transactions can still be processed when a printer is not working. The transaction processing procedure remains the same, but a Card Transaction Record Voucher (CC595) is completed as a substitute record of the transaction.

      An 'Approved' transaction will display a six digit reference number on the terminal screen. This number must be recorded on the voucher.

      Give the customer a copy of the voucher as their record of the transaction. The copy is dealt with as a normal printer receipt

      A 'Declined' transaction cannot be accepted. A Card Transaction Record may be issued at customer request.
Cumulative Totals Printout
Cumulative Totals Printout
    A Totals printout shows the total of EFTPOS transactions split into:
    • Purchases
    • Cash-Outs
    • Refunds

    The cumulative total displays the totals from the start of the days trading.

    Note:Totals are cleared to zero upon the automatic overnight reconciliation
