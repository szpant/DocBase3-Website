Queensland Compliance Information System (QCIS)
What it isThe Queensland Compliance Information System (QCIS) is used by Transport Inspectors (TIs) to record intercepts and undertake compliance actions.

QCIS allows TIs to search different databases and improves access to real-time information. QCIS allows TIs to issue the following by email (with the customer’s consent):
  • Infringement notices (individuals only)
  • Weighing device notices
  • Formal warning notices
  • Property receipts

All other notices must continue to be issued manually. If a Defect or Prohibition notice is issued during an intercept, the Defect Number and Severity must be recorded in QCIS.

QCIS can be accessed via either a mobile application or a desktop portal.

TIs access the app using an iPad mini while they are conducting on-road duties. The iPad mini connects to QCIS via a SIM card or WiFi.

TIs, Senior Transport Inspectors (STIs) and Managers can also access the QCIS Portal via their laptops or desktops while in the office.

When to use the NotebooksQCIS can be used to record details of the intercept process however if any non-compliance is detected sufficient details must be recorded in your official notebook to prove all the elements of an alleged offence and enable the Transport Inspector to recall the full facts during any subsequent court proceedings.

Details such as the vehicle, person, issue and compliance actions must be captured in the official notebook, as this is the record that may be provided as the record or evidence in any court scenario.

A notebook should not be viewed as a duplication of work, but a corroboration of work.

App
App
    The QCIS app allows TIs to complete a search in TICA and NEVDIS through one user interface, record intercepts and generate notices.

    TIs use the app to:
    • create intercepts and sightings (also known as footprints)
    • select and record compliance actions for the intercept they create
    • use standard tablet features such as the camera and GPS to assist with data capturing and reporting
    • use their digital signatures. The digital signature can be captured and uploaded into the app when the TI sets up their account.
    • obtain a signature from the customer.

    The app can:
    Pre-populate a customers eContact details
    Pre-populate a customers eContact details
      The app can utilise a customers email address if they have their eContact details recorded in TICA. On the Record Intercept (Service) screen, the registered email address will be pre-populated only for Queensland drivers.
    Electronically send notices
    Electronically send notices
      The app can provide customers with electronic notices via email, only if the customer provides their consent.
    Automatically update
    Automatically update
      The app has been designed to auto-update when a new version is made available. In order for this to occur you will need to be connected to wifi. It is an Apple requirement that a device must be connected to wifi in order to begin an iOS update. If wifi is not available, TIs should ‘hot spot’ from another TMR corporate device (iPad mini or mobile phone). Although this is still using the TMR data, Apple treats it the same as wifi.

      If the update fails to automatically install, TIs will need to access the app catalogue and manually trigger the update. If this also fails, TIs will need to delete the app from the device and download it again from the TMR App Catalogue.

      You should always ensure you have the latest version of the operating system available.
    Capture a customers signature
    Capture a customers signature
      Customers are able to sign for accountable forms on the iPad mini, however, there is no legal requirement for customers to sign for notices. The signatures captured through this process does not authorise the electronic issue of any other notices, and will not be stored in the Staff User Interface. The customer’s signature will not be used by the department for any other purpose.

      Capturing a signature is not mandatory, and the user will be able to issue the notice if the signature section is tapped on but not completed.

    The app cannot:
    Check Legislation
    Check Legislation
      The app is currently unable to check legislation directly, however you can utilise an internet browser on the iPad mini to access the required legislation.
    Display a Driver's photo (Facial Image)
    Display a Driver's photo (Facial Image)
      The app does not allow you to see a driver's photo (facial image) as restricted access to a person’s image is only permitted in certain circumstances under the Transport Planning and Coordination Act 1994 (TPC Act). A TI accessing a digital image to confirm a person’s identity is not a circumstance that is currently permitted.

      A range of amendments have been proposed to transport legislation, in particular with regard to sharing and use of information for a legitimate government purpose. The project team has been advised that the progress of this proposal is largely dependent on political will and other government priorities.

        Targeted operations
          Intercepts related to targeted operations with specific objectives and timeframes must be recorded using the relevant operation name.
          The name format is “Operation Name (Operation Type Code)”


          The Operation Type Code is:
          PERSPersonalised Transport (Taxi, Limo, Booked Hire or Booked Hire Rental)

          LVLight Vehicle
          HVHeavy vehicle

          An operation showing is QCIS as Road Safety Week (PERS) is a target operation for personalised transport vehicles ie Taxi, Limo, Booked Hire or Booked Hire Rental.

      Issuing an infringement
      Issuing an infringement



        Penalty Infringement Notices (PIN) can only be issued from QCIS during an intercept. If an investigation into an alleged offence is required as a result of an intercept, a case must be created in CID and the intercept is the trigger for the investigation. If the alleged offence is proven a hand written PIN may be issued to the responsible party.

        Infringements can only be issued to individuals from QCIS and must not be issued to Organisations or Corporations.

        Infringement details such as the Infringement Code, Associated Demerit Points and Fees will be pre-populated from the App as soon as the offence is selected.
        • Additional Information such as Number of Passengers and whether the licence has been produced is recorded.

        Infringements can be issue via email or post in the QCIS app:
        • If customer chooses to have their infringement emailed to them a field will appear where their email address can be selected, or entered if it has not pre-populated.
        • If the customer decides to have their infringement mailed then their address will be pre-populated from TICA. This address can be updated if required.


          Note:Changing contact details in the App will not update the details in TICA.


          Note:You can change the date of the offence in the QCIS App when issuing an infringement. An example of when you would do this is, you have found a fatigue offence in a work diary that occurred earlier in the month. Further details about how to change the date of offence can be found in the Issue Infringement Notice - Procedures.

        Infringements issued through QCIS are recorded in TICA and will have a C prefix.

          Note:In the event an infringement is submitted in QCIS and needs to be recalled,
          Refer: Penalty Infringement Notice Guide > Waiving a PIN

        Invalidating an intercept
        Invalidating an intercept
          An invalidate intercept will have a status of ‘invalid’ and can be searched via the app or portal using the 'Search for a Person or Vehicle' function. An invalid intercept will not be able to be reversed.

          A mandatory 'reason' must be selected, to invalidate an intercept.

          Note:Only Senior Users and Administrators are able to invalidate an intercept.


        Any change (edit or invalidate) made to an intercept will be recorded in an audit history log. This will be accessible through the Audit History Log tab, only available on the portal intercept record.

      Future features and things to know about Phase 1The below sections detail features which are still being prioritised for future QCIS releases and key things you need to be aware of when you’re using the QCIS App.
      OSOM Weighing
      OSOM Weighing
        The OSOM weighing feature is currently under review. Please follow the current procedure for weighing these vehicle combination configurations (manually-issued WDN’s). The Mass tab in the QCIS app can be used for standard configurations.
      Company/Organization CRN search
      Company/Organization CRN search
        While vehicle searches will return organisation details, there is currently no function to search for an organisation by name or CRN in QCIS. Only an individual can be searched and issued associated QCIS notices. The benefit of having this functionality is considered to be high and it will be incorporated in a later release. If additional organisation details are required they should be retrieved from TICA. Manual notices should be issued to organisations if required.
      Defect Notice History
      Defect Notice History
        A vehicle with a currently issued Road Patrol Defect will show a warning flag to the user, however only limited details are returned. TICA must be utilised for further information if it is required.
      COI (Certificate of Inspection)
      COI (Certificate of Inspection)
        COI expiry warnings may be displayed to the user. Further investigation via TICA is recommended to ensure accuracy of the expiry period or if there is an EOT (Extension of Time) in place.
      Static Sites
      Static Sites
        Some of the static sites shown in the app are currently incorrect – if you are using the static sites ensure the address displayed is correct. If not use the pin drop functionality to record the correct location.
      Issue with Alphanumeric Interstate Licence Numbers
      Issue with Alphanumeric Interstate Licence Numbers
        There is currently an issue with submitting an intercept if the persons licence number contains letters and numbers. The issue is being investigated.

        If a search is performed on an interstate licence and an error message is returned you will need to create a new person and enter all details with the exception of the licence number. This will allow you to submit the intercept without getting an error message.

        Your STI should be kept informed of how often this error is encountered, so that the fix can be prioritised accordingly.

        To create a new person Refer above: Using the QCIS > Search for a Person > Search for a Person Procedure
      Vehicle Description
      Vehicle Description
        The vehicle description shown in QCIS is taken from the Makes and Models section of TRAILS, which shows date of manufacture/build and compliance date.

        In TICA there is the option to complete a VIN decode, which provides general details of the vehicle at time of manufacture excluding colour, build date or seating. Alternatively vehicle details in TICA show the specific details of the vehicle including colour, build date and seating.

        The description available in TICA would be better for TIs, making this available will be considered for future development.
