AIS DMS Capture Procedure (7034)

Applies to
Applies to
    This procedure will outline how to capture AIS Compliance Checklist and supporting documentation, Finalised Corrective Actions and Show Cause Notices in the TMR Document Management System (DMS).
Before you start
Before you start
    The Accreditation Unit create a DMS container for each AIS when it is approved.

    You will require access to Road System Management (RSM) AIS DSM containers.

    All documents that are to be attached in DMS containers (specific DMS number is recorded on RUMIIS) must be electronic. Records cannot be saved to the DMS safety net.
1
Scan all documents and save with the file name to include the AIS name and number.
2
Note the DMS container number (540/XXXXX) located in RUMIIS against the AIS
3
Open DMSV8 RSM application on IBM Notes Workspace.
Refer: File > Open > IBM Notes application > look for server - shb1ln16/a/qdot/au > DMS > Road System Management
4
Select 'Capture an Item' from the main menu of DMS.
5
Complete the profile. Fields with a red asterisk * are mandatory and must be completed with below content:
  • Item Type: Audit
  • Item format: Electronic
  • Circulation: Internal
  • Subject: Audit Report and Finalised CAs – <enter AIS Station Name> – <enter AIS Number>
6
Select Container: click on select container – click on container ID – type in DMS container number and enter.

Requested container will appear at the top of the list.

Click on OK button
7
Document Contents: attach the final version of your electronic item via the Lotus Notes paper clip option or drag and drop from the Windows Explorer window.
8
Select 'Capture'.
9
A pop-up will appear with the Item ID that has been assigned.
