Report Access Portal (RAP) (7187)


What it is?The Report Access Portal (RAP) is an online reporting tool which can be used by management and senior staff for operational performance, planning and review.
When is it used?The RAP reports can be used to:
  • Analyse individual/team/centre performance
  • Identify trends/fraud/security breaches
  • Provide statistical information to assist with rostering/planning etc
RAP LinkReport Access Portal
ReportsRAP contains a suite of reports which can be utilised by authorised staff when needed. Below is an introductory list of the reports that officers new to RAP may find useful.
CSC Management Reports
D-140 - QMatic: Daily wait time for last 6 business days
D-140 - QMatic: Daily wait time for last 6 business days
    This report is used to review the number of customers and their wait times for the last 6 business days. If the wait time exceeds the CSB target, utilise other reports (such as W-266 QMatic Hourly Segmentation Analysis) and consider external factors (sick leave/training) to determine why.

    Report example and more information.

    D140 Report Example

    RAP navigation path: 2. Q-Matic > 1. All Summary Reports > 02. Summary Detail
W-266 - QMatic: Hourly Segmentation Analysis
W-266 - QMatic: Hourly Segmentation Analysis
    This report shows four different hourly segmentations which can be used in conjunction with D-140 (see above) to assess the previous day's performance.

    Report example for more information.

    W266 Report Example

    RAP navigation path: 2. Q-Matic > Select CSC location > 01. Segmentation Analysis
M-306 - QMatic: Average wait and service segmentation
M-306 - QMatic: Average wait and service segmentation
    This report is for local management teams to review (recommended on a quarterly basis) to ensure sufficient resources are allocated/rostered according to wait time standards.

    Over a 6 month period it will show the average wait time and average number of counters open for each week day at half hour intervals. This can be used to determine whether shift times/rosters need to be amended.

    Any changes implemented would not be visible on this report until the next 6 monthly check. Management could utilise the W-219 report (refer below) to check weekly progress following amendments being made.

    Report example for more information.

    M-306 Report Example

    RAP navigation path: 2. Q-Matic > 1. All Summary Reports > 01. Wait time trends > M-306 (excel)
W-219 - QMatic: Weekly Trend Overview
W-219 - QMatic: Weekly Trend Overview

    This report can be run to identify any trends that are occurring weekly or to check on the progress of changes implemented.

    Report example for more information.

    W219 Report Example

    RAP navigation path: 2. Q-Matic > Select CSC > 02. Weekly Trend Overview
QMatic Matter Comparison Report (Bubble Report)
QMatic Matter Comparison Report (Bubble Report)
    This report gives an overview of how CSCs are performing compared to other CSCs and how their individual operators are performing.

    When generating this report, consider that figures for the previous month are not updated until the 10th of the next month.

    Refer: Operator Management Reports > QMatic Matter Comparison reports) for more information on operator performance.

    Report example for more information.

    QMatic Matter Comparison Report Customer Management

    Report navigation path: 2. Q-Matic > 1. All Summary Reports > 06. Matter Comparison
Operator Management Reports
QMatic Matter Comparison Reports (Bubble Report)
QMatic Matter Comparison Reports (Bubble Report)
    This report gives an overview of how CSCs are performing compared to other CSCs and how their individual operators are performing.

    When generating this report, consider that figures for the previous month are not updated until the 10th of the next month.

    It compares the transaction time for each matter code (not each customer) so if an operator serves one customer for two transaction types they should have recorded a transaction time for each (e.g. 1 for driver licence and 1 for registration).

    Refer to the below report example for more information.

    QMatic Matter Comparison Reports Operator Management

    Report navigation path: 2. Q-Matic > 1. All Summary Reports > 06. Matter Comparison > Select a location > Select month/s *hold the Ctrl key to select more than one month* > Click Finish to run report
Customer Service Officer Performance Report
Customer Service Officer Performance Report
    This report displays transactional performance of an operator on a counter and can be used for feedback, identifying training needs and locating issues with the operation of QMatic. The user's data should be consistent with or better than the state averages.

    When delivering feedback, consideration should be given to:
    • external factors such as dealer work, counter hopping, partial days, training, buddying and the business mix of the CSC.
    • state averages (if both the state and operator have improved the improvement may not be apparent).

    Refer to the below report example for more information.

    Customer Service Officer Performance Report

    Report navigation path: 1. Client Service Delivery > 6. CSO Counter Performance
QMatic Matter Codes vs TICA Transactions
QMatic Matter Codes vs TICA Transactions
    This report uses data that CSO's enter into QMatic in conjunction with what is displayed and updated in TICA. It can be used along with the CSO Counter Performance and Ticket Validity Reports to ensure the operators are using QMatic correctly.

    Incorrect matter codes impact other reports and the average transaction times, therefore it is imperative that correct matter codes are entered into QMatic and multiple matter codes are entered when applicable.

    For more information or training about matter coding refer to the following which are located in Report Access Portal:
    • QMatic Standard Procedures Operator Manual and Wed Terminal Guide
    • QMatic Standard Procedures Managers and Supervisor Manual

    Refer to the below report example for more information.

    QMatic Matter Codes vs TICA Transactions Report

    Report navigation path: 2. Q-Matic > 1. All summary reports > 06. Matter Comparison > 02. QMatic matter codes vs TICA transactions
Ticket validity reporting - Daily detail for specified location
Ticket validity reporting - Daily detail for specified location
    This report shows the previous 4 days ticket validity and can be used to help determine if QMatic is being used correctly. Each ticket iteration (including invalid tickets and their reason) will be displayed.

    Refer to the below report example for more information.

    Ticket Validity Reporting

    Report navigation path: 1. Client Service Delivery > 2. Q-Matic > Select Appropriate CSC > 04. Ticket validity reporting
