Smartcard Production Status (3360)

Production status
Production status
    The smartcard TICA history screens will contain a different status depending on the stage of the production, as displayed in the table below.
    Smartcard Status Definitions
    Captured A smartcard transaction has been started with the smartcard information recorded but the transaction has not been completed.
    Note:The status is visible for lodged IA applications when the fees are not required.
    • If the application is approved, the smartcard status will move to registered.
    Pending A transaction requiring a smartcard fee has been processed, but the payment has not yet been receipted, or the payment has been unmatched/cancelled.

    Refer to Pending Smartcard Status section below for transactions that can be processed while the smartcard is in a pending status.
    Paid An Industry Authority application requiring a smartcard fee has been lodged and receipted but not yet approved.

    Note:If the application is approved, the smartcard status will move to registered.
    Registered The application has been approved and is ready for a 1:N match, or if none required, ready to be sent to the production house.

    Refer to Registered Status section below for transactions that cannot be processed while the smartcard is in registered status.
    Production Requested The smartcard is being made by the production house.

    Refer to Production Requested Status section below for transactions that cannot be processed while the smartcard is in production requested status.
    Active The smartcard has been produced and is being/has been mailed to the customer.
    Expired The physical smartcard has expired (not necessarily the product).
    Cancelled A smartcard is cancelled for any of the following reasons:
    • It has been recorded as returned in TICA with a non return reason eg. lost, stolen or destroyed.
    • 28 days has elapsed since a transaction was recorded that required the non immediate return of the smartcard.
    • An Active smartcard has been invalidated by the IMU.
    Returned The smartcard has been physically returned and the return has been processed in TICA (includes faulty/defective).
Pending Smartcard status
Pending Smartcard status
    Transactions that can still be performed on a pending smartcard, up to and including the status of 'Registered' include:
    • Address changes
    • Update customer information
    • Invalidate information
    • Add MLI code (driver licence only).
Registered status
Registered status
    The following transactions are prevented:
    • Cancellation of all products
    • Surrender (driver licence, marine licence, industry authority, photo identification card)
    • Amend industry authority
Production Requested status
Production Requested status
    Some updates/amendments to the details/information on the smartcard cannot be processed.

    The following transactions are prevented:
    • Correction of smartcard product details
    • Downgrade of a driver licence, where the downgrade action requires a smartcard to be returned
    • Surrender of a driver licence, marine licence, IA or APA/Photo ID
    • Replacement smartcard
    • Change of name
    • Change of customer details
    • Amend industry authority
    • Merging of customer details where one customer has a smartcard in this state
    • Unmatch a smartcard, licence or adult proof of age payment
    • Invalidation of a transaction that involves a smartcard request
    Note:The card must be 'Active' for the processing of these transactions as the return of the smartcard may be required. Refer to: Smartcard Guide > Smartcard Return.
