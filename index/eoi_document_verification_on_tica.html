EOI Document Verification on TICA (5010)


EOI Document Verification on TICA
What it isIn an effort to minimise identity fraud, TMR records the details of EOI, Name Change and Residency documents presented by a customer against their record on TICA. The documents may then undergo duplicate matching, TICA product matching and/or external verification check.
Documents CapturedThe following documents presented are to be recorded on TICA:
~ Category A
~ Name Change 		~ Category B
~ Evidence of Residency
Any First or Second Level Exemption used is also recorded.
External Verification
Documents that undergo External Verification
Documents that undergo External Verification
    Initially the following documents will be verified:
    • NEVDIS interstate driver licence (excluding external territories)
    • Birth Certificates
    • Australian Passports
    • DIBP Immicards
    • Australian Citizenship certificates (excludes Australian Citizen by Descent certificates)

    Other authorities will be added as this service becomes available.

      Duplicate Matching
      Documents that are Duplicate matched
      Documents that are Duplicate matched
        All category A documents and some name change documents recorded in TICA will automatically undergo a duplicate check to determine if there is another identity document recorded for a different customer that exists. If so, a document investigation will follow.

        Allowable Duplicate Documents
        Allowable Duplicate Documents
          Some documents are permitted to be recorded for more than one Customer. If there is already an allowable document associated with a customer, the new customer shall be permitted to be associated with the same allowable document.
          • Australian Citizenship Certificate;
          • Dept. of Immigration and Border Protection (DIBP) (formerly Dept. of Immigration and Citizenship - DIAC) document for travel to Australia and;
          • Marriage Certificate (recorded for up to two customers)

          For these documents - more than one customer may be associated with the document when the document is correctly 'updated' to reflect that another customer is also applicable to (i.e. named on) it, then the document is not considered a duplicate.

          Refer: Customer > 1. Customer > The How To: Customer > Record/Update/Exempt /Invalidate EOI, Name Change, Residency Document or Exemption > Record EOI/Name Change/Residency Document or Exemption (TICA) > 3. Record additional document details > Document is legitimately used by more than 1 customer

        Maximum character length
        Maximum character length
          DVS accepts 60 characaters for a given name and 50 characters for a surname for all state birth certificates except Victoria. Victorian certificates are allowed 80 characters for both given and family names.
      Australian Passport or travel document
      Australian Passport or travel document
        Single Name
        Single Name

        Date of Birth
        Date of Birth

          Date of birth is entered using numerical values in the range:
          Day range 0-31
          Month 0-12
          Year 1753-3000
        Name exceeds character length
        Name exceeds character length
          In DVS the maximum length of each given and surname is 31 characters. If a customer’s given or surname exceed this limit you should record as many full names as possible, any other names that don’t fit should be represented by their initials (without full stops.)

          For example:
          Customer's surname is Krishnamurty Don Sariyaarachchige Krishna it should be recorded as Krishnamurty Don S K.

        Single Name

        Maximum character length

      Citizenship Certificates
      Citizenship Certificates
        Single Name
        Single Name
          Single name certificates must be entered into DVS with the given name blank and single name in the family name field
        Date of Birth
        Date of Birth

        Maximum character length

        Stock Number
        Stock Number
          The stock number may be located on the front or the back of the certificate. The full stock number, both alpha and numeric, must be entered into DVS.

          For example; a Citizenship Certificate with a Stock No displaying AC(G) No. 12345 is to be recorded in TICA as ACG12345. Do not enter the () or the "No.".

        Exceptions
        Exceptions
          Interstate Driver Licence fails NEVDIS Verification
          Interstate Driver Licence fails NEVDIS Verification
            If an interstate Driver Licence for a jurisdiction that is on NEVDIS fails verification the CSC/Agency is required to manually verify the document using the fax or email procedure outlined for Driver Licence transfers.

            Refer: Learner or Provisional/Probationary or Open Licence Guides > Transaction Details > Driver Licence Transfers > Confirming Non Qld Details

            Once the fax/email confirmation has been received from the interstate authority, the CSC/Agency will need to fax/email the confirmation onto the IMU to confirm and update the document status.

          Off-Line Transfer of Registration
          Off-Line Transfer of Registration
            If there is a problem verifying the acquirer's EOI for a vehicle or vessel registration being processed at a CSC from an off-line agency, the following action is to be taken:
            1.EOI Document is to be referred to the IMU for investigation.
            2.EOI Document record in TICA may be invalidated and a SLEN recorded.
            This action will allow the transfer transaction to be completed thereby protecting the seller.
            3.If IMU verifies document, the EOI is to be re-added to the customer's record.
            Note:If after investigation the document is found to be suspect show cause action will need to be taken.
